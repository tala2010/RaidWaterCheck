团本划水检测 Raid Water Check 插件讲解稿

一、开场介绍

大家好，今天介绍一个魔兽世界团本复盘插件，名字叫“团本划水检测”，英文名 Raid Water Check。

这个插件不是用来吵架的，也不是用来给谁直接定罪的。它的定位是团本表现复盘工具，主要给团长、指挥、固定团管理使用。它会在 Boss 战结束后，自动整理一份团队表现报告，帮助我们快速看到谁可能需要复盘，比如有没有长时间无动作、有没有死亡、有没有吃可规避伤害、有没有打断驱散、有没有使用关键技能等等。

插件默认只记录 Boss 战，不会一直记录小怪。如果你想记录小怪或者普通战斗，也可以在设置里打开。


二、插件入口

插件安装好以后，游戏里可以用两个命令打开：

/rwc
/huashui

也可以通过右上角插件集合图标或者小地图按钮打开。

小地图按钮的操作是：

左键：打开上一场战斗报告
右键：打开设置面板
拖动：移动按钮位置


三、报告面板怎么用

打完 Boss 后，如果开启了自动弹出报告，插件会自动显示本场战斗统计。

报告里主要有这些列：

玩家：团队成员名字
状态：根据分数显示正常、略异常、需复盘、高疑似
分数：综合表现分，分数越低越需要复盘
伤害：本场造成的伤害
治疗：本场有效治疗
承伤：本场受到的伤害
可规避：吃到自定义可规避伤害的次数
施法：成功施法次数
打断：打断次数
驱散：驱散或偷取次数
死亡：死亡次数
原因：插件给出的异常原因

点击某个玩家，可以打开个人详情页。

个人详情页会显示：

主要伤害技能
主要治疗技能
施法次数
主要承伤来源
可规避伤害来源
死亡、打断、驱散、异常原因等汇总

这样我们就不是只看一个 DPS 排名，而是能看到这个人到底是低输出、死得早、没打断、吃机制，还是因为分工导致数据低。


四、插件判定原理

这个插件的核心原理是读取魔兽世界插件 API 提供的战斗日志事件，也就是 Combat Log。

战斗日志里会出现很多事件，比如：

谁对谁造成了伤害
谁进行了治疗
谁成功施放了技能
谁打断了技能
谁驱散了效果
谁死亡了
谁受到了某个技能伤害

插件会在 Boss 战开始时清空统计，在 Boss 战结束时生成报告。

它主要从以下几个维度判断是否异常：

1. 死亡次数

如果一个人在战斗中死亡，插件会扣分，并在原因里显示死亡次数。

死亡本身不一定代表划水，因为有可能是战术安排、治疗压力、坦克换嘲问题，或者团队机制失败导致。但死亡一定是复盘时需要看的重点。

2. 长时间无动作

插件会记录每个人最后一次有效动作的时间。

有效动作包括：

造成伤害
治疗
施法成功
打断
驱散

如果一个人在战斗中超过设置的宽限时间没有任何有效动作，插件会累计无动作时间。

比如设置里无动作宽限是 6 秒，如果某个人 20 秒没有任何有效动作，那么插件会把超过宽限的部分计入无动作时间。

战斗结束后，如果无动作比例超过轻度、中度、重度阈值，就会扣分并显示“无动作百分比”。

这个维度主要用来发现明显挂机、发呆、长时间不打不奶的人。

3. 施法频率

插件会统计施法成功次数，并计算每分钟施法次数。

如果一个人有输出或治疗数据，但施法频率明显偏低，就会扣分。

这个指标适合辅助判断一个人是不是按键频率太低，或者是不是长时间没有参与战斗。

需要注意的是，不同职业和专精的施法频率不一样，所以这个指标只是辅助，不是绝对判定。

4. 贡献占比

插件会统计全团总伤害和总治疗，再看每个人的贡献占比。

如果某个人的伤害或治疗占比过低，并且战斗时间足够长，插件会提示贡献偏低。

但是这个指标也要结合实际情况，比如：

有人负责跑机制
有人负责转火
有人死得早
有人是工具人职责
有人装等低
有人是坦克或治疗

所以贡献低不等于一定划水，只是提醒团长需要看一下。

5. 承伤占比

插件会统计每个人受到的总伤害。

如果一个人的承伤占全团承伤比例过高，插件会提示承伤异常。

这个可以帮助发现：

总是站错位置的人
总是吃地板技能的人
坦克以外异常承伤的人
机制处理有问题的人

不过坦克本来就承伤高，所以看这个指标时要结合职责。

6. 可规避伤害

可规避伤害不是插件自动知道的，需要我们自己添加技能 ID。

比如某个 Boss 有地板火、正面喷吐、点名圈、爆炸圈，这些技能我们认为是应该躲开的，就可以添加成可规避技能。

命令示例：

/rwc avoid add 123456 地板火

添加以后，如果之后战斗里有人吃到这个技能，插件就会记录为可规避伤害，并根据设置扣分。

查看当前规则：

/rwc rules

删除规则：

/rwc avoid del 123456

这个功能适合固定团长期使用。打一个本的时候，边打边把常见机制技能 ID 加进去，后面复盘就会越来越准。

7. 关键技能缺失

插件支持自定义关键技能检查。

比如团长希望骑士一场战斗至少开一次复仇之怒，法师至少用一次时间扭曲，德鲁伊应该用战复，治疗应该交团队减伤，都可以添加成关键技能。

命令示例：

/rwc track add 31884 复仇之怒 PALADIN 1
/rwc track add 80353 时间扭曲 MAGE 1

格式是：

/rwc track add 技能ID 技能名 职业英文 最低次数

职业英文比如：

PALADIN 圣骑士
MAGE 法师
DRUID 德鲁伊
PRIEST 牧师
WARRIOR 战士
ROGUE 潜行者
HUNTER 猎人
SHAMAN 萨满
WARLOCK 术士
DEATHKNIGHT 死亡骑士
MONK 武僧
DEMONHUNTER 恶魔猎手
EVOKER 唤魔师

如果某个职业该用的关键技能没有用，插件会在原因里显示“未使用关键技能”。


五、综合分数怎么来的

插件每个人默认从 100 分开始。

然后根据异常项扣分，比如：

死亡会扣分
无动作比例过高会扣分
施法频率过低会扣分
贡献占比过低会扣分
承伤占比过高会扣分
吃到可规避伤害会扣分
缺失关键技能会扣分

最后得到一个综合分数。

分数越低，代表越需要复盘。

插件默认会把人分成几个状态：

正常
略异常
需复盘
高疑似

这里要强调：分数不是判决书，只是复盘优先级。真正判断一个人是不是划水，需要结合职业、专精、机制分工、战斗日志和团队安排。


六、设置面板怎么讲

设置面板分成两个页签：

基础设置
高级设置

基础设置里主要是常用开关：

战斗结束后自动弹出报告：
打开后，Boss 战结束自动显示报告。

也记录小怪/非Boss战：
默认关闭。关闭时只记录 Boss 战；打开后小怪战和普通战斗也会记录。

最短记录战斗时长：
太短的战斗会被忽略，避免误触发。

报告显示人数：
控制报告列表最多显示几个人。

通报分数线：
点击“通报异常”时，只会通报低于这个分数线、并且有异常原因的人。

扣分开关：
可以单独开关施法频率、贡献占比、承伤占比、可规避伤害、关键技能缺失等扣分项。

高级设置里主要是阈值和扣分权重：

无动作宽限时间
轻度无动作阈值
中度无动作阈值
重度无动作阈值
高承伤占比阈值
每次可规避伤害扣分
缺失关键技能扣分

如果是普通团队，默认设置就可以用。

如果是固定团，可以根据团队要求调严一点。


七、通报异常

报告面板里有“通报异常”按钮。

点击以后，插件会把异常名单发到团队或队伍频道。

现在通报规则是：

分数低于通报分数线
并且有明确异常原因

也就是说，插件不会只因为分数低就乱通报，必须有原因，比如死亡、无动作、可规避伤害、缺失关键技能等。

这也是为了避免刷屏和误伤正常玩家。


八、使用建议

建议团长这样用：

第一步，先默认只记录 Boss 战。

第二步，打完 Boss 后看报告里分数最低的几个人。

第三步，点开个人详情，看他具体异常是什么。

第四步，如果是机制问题，把对应技能 ID 加到可规避伤害规则里。

第五步，如果团队有固定要求，比如嗜血、战复、团队减伤，就把这些技能加到关键技能规则里。

第六步，打几次以后，规则会越来越适合自己的团队。


九、重要注意事项

这个插件不是官方评分，也不是 Warcraft Logs 的替代品。

它适合快速复盘和团内管理，但不要只凭一个分数就骂人。

以下情况都可能导致数据看起来异常：

特殊机制分工
跑位多
转火多
提前死亡
装等低
职业弱势
治疗分工不同
坦克承伤天然高
战斗日志 API 限制

所以最正确的用法是：

插件帮你找重点，团长再结合实际情况判断。


十、结尾引导

如果你是团长、指挥，或者经常打固定团，这个插件可以帮你快速看出战斗里哪些地方值得复盘。

它不会替你做判断，但能帮你省下翻日志和逐个问人的时间。

后续还可以继续加更多功能，比如 Boss 技能规则库、职业专精模板、团队减伤时间轴、可规避伤害排行榜等等。
